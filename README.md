# StatementGraph — Анализ банковских выписок

Веб-приложение для автоматического анализа банковских выписок по кредитным/дебетовым картам. Загружает PDF-выписку, парсит транзакции, нормализует данные, выявляет подозрительную активность и визуализирует связи между транзакциями в виде интерактивного графа.

## Возможности

- **Загрузка PDF-выписок** — drag-and-drop загрузка, автоматическое извлечение метаданных (банк, номер карты, период)
- **Парсинг и нормализация** — извлечение транзакций из PDF (pdfplumber), нормализация мерчантов, классификация операций
- **Скоринг и детекция аномалий** — 8 типов алертов: дубликаты, всплеск мерчанта, возвраты, аномалии сумм, задержки расчётов, гигиена мерчантов, динамика долга, аномалии регулярности
- **Интерактивный граф** — визуализация связей транзакций (Cytoscape.js): мерчанты, группы, алерты, карты
- **ИИ-анализ** — аналитическое заключение по выписке через локальную Ollama (DeepSeek v3.1)
- **Управление алертами** — фильтрация по критичности/типу, сортировка, смена статуса, экспорт CSV
- **Сводка** — статистика: обороты, количество алертов, топ рискованных мерчантов, детали возвратов и дубликатов, экспорт CSV
- **Экспорт данных** — выгрузка алертов и сводки (возвраты, дубликаты) в CSV-файлы

## Технологический стек

### Бэкенд
- **Python 3.13+** / **FastAPI** — REST API
- **SQLAlchemy 2.0** + **PostgreSQL 16** — хранение данных
- **Neo4j 5.23** — графовая БД для визуализации связей
- **pdfplumber** — парсинг PDF
- **scikit-learn** — Isolation Forest для обнаружения аномалий
- **NetworkX** — построение графов в памяти
- **Ollama** — интеграция с локальными LLM

### Фронтенд
- **React 18** + **TypeScript** — UI
- **Vite 5** — сборка
- **TanStack Query v5** — управление серверным состоянием
- **Cytoscape.js** — интерактивная визуализация графов
- **React Router v6** — навигация

### Инфраструктура
- **Docker Compose** — оркестрация 4 сервисов
- **GitHub Actions** — CI (lint, typecheck, тесты)

## Быстрый старт

### Предварительные требования
- Docker и Docker Compose
- (Опционально) Ollama для ИИ-анализа

### Запуск

```bash
cd StatementGraph
cp .env.example .env
docker compose up -d
```

Приложение будет доступно:
- **Фронтенд:** http://localhost:5173
- **API:** http://localhost:8000/api/v1
- **Neo4j Browser:** http://localhost:7474

### Использование

1. Откройте http://localhost:5173
2. Загрузите PDF-выписку (drag-and-drop или клик)
3. Нажмите «Запустить анализ» — пайплайн: загрузка → парсинг → нормализация → скоринг
4. Перейдите на вкладку «Сводка» — общая статистика и топ рискованных мерчантов
5. Вкладка «Алерты» — детальный просмотр каждого алерта с рекомендациями
6. Вкладка «Граф» — интерактивная визуализация связей между транзакциями

## API

| Метод | Эндпоинт | Описание |
|-------|----------|----------|
| POST | `/statements/upload-pdf` | Загрузить PDF-выписку |
| POST | `/statements/{id}/parse` | Парсить транзакции из PDF |
| POST | `/statements/{id}/normalize` | Нормализовать данные |
| POST | `/statements/{id}/score` | Запустить скоринг алертов |
| GET | `/statements/{id}/summary` | Сводка по выписке |
| GET | `/statements/{id}/ai-analysis` | ИИ-анализ (Ollama) |
| GET | `/alerts` | Список алертов (фильтр по severity) |
| PATCH | `/alerts/{id}` | Обновить статус алерта |
| GET | `/transactions` | Список транзакций |
| GET | `/graph/alert/{id}` | Граф по алерту |
| GET | `/graph/merchant/{name}` | Граф по мерчанту |

## Типы алертов

| Тип | Описание |
|-----|----------|
| Дубликат | Повторная транзакция (та же сумма, тот же мерчант, < 30 мин) |
| Всплеск мерчанта | 3+ транзакции у одного мерчанта за 2 часа |
| Возврат | Зачисление, совпадающее по сумме (±10%) с предшествующим списанием |
| Аномалия суммы | Сумма > 3 медиан у данного мерчанта |
| Задержка расчёта | Задержка расчёта > 72 часа |
| Гигиена мерчанта | Мерчант < 3 транзакций + сумма > 5000 |
| Динамика долга | Рост задолженности > 50% за одну операцию |
| Аномалия регулярности | Аномальный интервал в регулярных платежах |
